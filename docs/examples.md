# Diagnostic Examples

This document provides examples of logs generated by the Hydration Consistency Checker and the Memory Snapshot Logger.

## 💧 Hydration Mismatch Examples

Hydration mismatches occur when the server-rendered HTML differs from the initial client-side render. 

**Example Log entry:**
```json
{
  "type": "hydration",
  "message": "Hydration mismatch detected!",
  "data": {
    "expected": "12450",
    "actual": "12448",
    "path": "/clusters"
  },
  "timestamp": 1714214400000
}
```
*Note: In this system, mismatches are tracked by comparing the string length of the JSON data passed from the server vs the data received by the client component.*

## 🧠 Memory Snapshot Examples

The system records JS heap usage on every route transition and warns when a significant delta is detected.

**Example Warning Log:**
```json
{
  "type": "memory",
  "message": "Significant memory change on route /details/42",
  "data": {
    "delta": "14.22 MB",
    "before": "45.10 MB",
    "after": "59.32 MB"
  },
  "timestamp": 1714214500000
}
```
*Note: A 10MB threshold is used to filter out noise from minor garbage collection cycles. Tracking uses the `performance.memory` API in supported browsers.*
